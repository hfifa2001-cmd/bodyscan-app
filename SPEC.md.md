## 0. 한 줄 목표
사용자가 쉽게 얻는 건강 데이터(검진 수치 · InBody · 전신 사진)로 개인화된 인체를 시각화하고, 현재 신체 상태를 자동으로 설명해 주는 웹앱.

## 1. 핵심 원칙 (정직성 · 범위)
- **외형 = 측정값 기반 추정**, **내부 장기 = 표준 모델 + 내 수치 시각화**, **자세 = 참고용 스크리닝.** 의료 진단이 아님(하단 면책 고지 고정).
- 사진 → 자세 분석은 **자동(MediaPipe Pose)** 이 기본. 수동 점 보정은 보조 fallback.
- 과장 금지: "내 장기 스캔"이 아니라 "내 데이터로 추정/시각화"로 표현.

## 2. 앱 구조 — 두 영역(탭) + 공통
- **탭 A: 근골격 · 체형** — 체형 입력 + 사진 자세 분석(정면/측면) + 과부하/약화 근육 맵(3D).
- **탭 B: 내과 · 장기** — 검진 수치 → 장기 색/이펙트 시각화.
- **공통(상단):** 성별 토글(남/여) — 체형/근육 모델에 영향.
- 각 탭마다 별도 리포트 패널 + AI 분석 버튼.

## 3. 입력 데이터 (3계층)
**계층1 · 체형:** 키(cm), 몸무게(kg), 성별, 나이, 체지방률(%), 골격근량(kg), 내장지방(1–20), 둘레(가슴/허리/엉덩이/허벅지/팔뚝/손목, cm)
**계층2 · 검진 수치:** ALT(간), LDL(혈관), 수축기혈압, 공복혈당, eGFR(신장), CRP(염증), 헤모글로빈(빈혈)
**계층3 · 자세(사진):** 정면/측면 전신 사진 → MediaPipe Pose 33 랜드마크 자동 추출

기본값(데모용 가상 검진 1인): 키174 몸무게82 남 38세 / 체지방24% 골격근32kg 내장지방11 / 허리92 엉덩이100 가슴100 / ALT52 LDL158 혈압138 혈당112 eGFR88 CRP0.6 Hb15

## 4. 자세 분석 (계층3) — 실제 구현
**랜드마크:** `@mediapipe/tasks-vision`의 PoseLandmarker(IMAGE 모드)로 정적 사진에서 33점 추출. 인덱스: 코0, 귀7/8, 어깨11/12, 골반23/24, 무릎25/26, 발목27/28.

**정면 지표(각도/비율):**
- 어깨 기울기 = `atan2(shL.y - shR.y, shR.x - shL.x)·180/π` (부호로 높은 쪽 판정)
- 골반 기울기 = 같은 식(hip23/24)
- 머리 기울기 = 같은 식(ear7/8)
- 좌우 비대칭 = |dist(shL,hipL) - dist(shR,hipR)| / 평균
- 측방 변위 = (어깨중심x - 발목중심x)
- 무릎 정렬 = (무릎중심x - 골반중심x) → 외반/내반
- 척추 측만 = [코, 어깨중심, 골반중심, 무릎중심] x좌표 표준편차 > 임계값

**측면 지표:**
- 전방 머리(거북목) = |귀.x - 어깨.x| / 체간길이
- 라운드숄더 = |어깨.x - 골반.x| / 체간길이
- 체간 기울기 = 어깨–골반 선의 수직 대비 각도
- 무릎 각도 = ∠(골반, 무릎, 발목) (반장슬/굴곡)
- 정렬 편차 = 귀/어깨/골반/무릎/발목 x의 분산

**근육 과부하/약화 규칙(임계값 예: 각도>3°):**
- 어깨 높은 쪽 → 상부승모근·견갑거근 **과부하**, 낮은 쪽 전거근·하부승모근 **약화**
- 골반 높은 쪽 → 요방형근 **과부하**, 반대 쪽 중둔근 **약화**
- 머리 기울기 → 동측 흉쇄유돌근 **과부하**
- 척추 측만 → 볼록 쪽 척추기립근 **과부하**
- 무릎 외반 → 장경인대·대퇴근막장근 **과부하**, 중둔근·내전근 **약화**
- 거북목(측면) → 상부승모근·흉쇄유돌근 **과부하**, 심부경추굴곡근 **약화**
- 라운드숄더(측면) → 대흉근·소흉근 **과부하**, 능형근·중하부승모근 **약화**
- 체간 전방 기울임(측면) → 척추기립근·장요근 **과부하**, 복근·둔근·햄스트링 **약화** (lower-crossed)

> 주: 위 규칙은 단순 스크리닝 휴리스틱. 코드에 임계값 상수로 분리해 조정 가능하게 할 것.

## 5. 시각화
- **체형 아바타(파라메트릭):** 키/체지방/근육/둘레 → 메쉬 변형. (프로토타입은 SVG morph; 본 사양의 목표는 아래 6의 3D)
- **장기:** 표준 장기 모델 위에 수치로 색/이펙트 — 간(ALT) 선홍→갈색, 심장(혈압) 박동 강도/색, 신장(eGFR) 투명도, 혈관(LDL) 플라크 개수, 염증(CRP) 붉은 아우라, 빈혈(Hb) 전체 채도.
- **근육 맵:** 과부하=빨강 / 약화=파랑, 앞/뒤 전환. **← 다음 단계에서 SVG가 아닌 실제 3D로 구현(아래 6).**

## 6. ★다음 단계 — Z-Anatomy 3D 연동 (이번 작업의 핵심)
**스택:** Three.js + GLTFLoader + OrbitControls (+ 필요 시 DRACOLoader).

**모델 준비(사람이 수동, 자동화 불가 — 사용자가 함):**
- Z-Anatomy(오픈소스, **CC BY 4.0**) Blender `.blend` 다운로드 → 근육(Muscular system)만 남기고 → glTF 2.0 `.glb`로 내보내기(Draco 압축은 끄는 게 호환에 안전). **출처 표기 필수.**
- 앱은 이 `.glb`를 입력으로 받음(파일 업로드 또는 정적 호스팅 URL).

**구현:**
- `.glb` 로드 → 씬 traverse → 모든 Mesh 수집(`{mesh, name, lname}`), 카메라 bounding-box fit, OrbitControls(회전/확대/이동).
- **근육 메쉬 이름(라틴명, 예 `Trapezius.l`)으로 하이라이트.** 매핑 테이블:
  ```
  BASE = { trap:['trapezius'], scm:['sternocleido'], pec:['pectoralis'],
    abs:['rectus abdominis'], psoas:['psoas','iliacus'],
    quad:['vastus','rectus femoris'], itb:['iliotibial','tensor fascia'],
    rhomb:['rhomboid'], erector:['erector','spinalis','longissimus','iliocostalis'],
    ql:['quadratus lumborum'], glute:['gluteus'],
    ham:['biceps femoris','semitendinosus','semimembranosus'] }
  ```
  좌/우는 메쉬 이름의 `.l`/`.r` 또는 `left`/`right` suffix로 판정.
- **핵심 연결 API:** `applyResult(overKeys, weakKeys)` — 4의 자세 분석이 만든 과부하/약화 키 목록(예 `['trapR','qlL']`, `['rhombL']`)을 받아 해당 근육 메쉬에 빨강/파랑 머티리얼 적용. **이 함수가 분석 로직과 3D 모델을 잇는 단일 접점.**
- UI: 메쉬 리스트(검색 + 클릭 시 색 순환 none→over→weak), 시점 리셋, 앞/뒤(또는 자유 회전).
- 모델 미보유 시 빈 상태 안내 + 변환 가이드 노출.

> 참고: 프로토타입 `anatomy3d-viewer.html` 가 이 6번의 작동 뼈대(로드 + applyResult)를 이미 담고 있음 — 이를 본 앱에 통합·확장할 것.

## 7. AI 리포트
- 입력 전체 + 자세 지표 + 과부하/약화 근육 목록 → LLM → JSON 반환:
  `{summary, riskFlags[], bodyShape, postureAnalysis, overloadMuscles[], weakMuscles[], top3Actions[]}` → 카드 렌더.
- 탭별 분리(근골격 리포트 / 내과 리포트).
- **실제 앱:** Anthropic API(`claude-sonnet-4-x`)는 **서버(백엔드)에서 호출**하고 API 키는 서버에만 보관(브라우저 노출 금지). 규칙 기반 로컬 리포트는 API 없이도 항상 작동(폴백).

## 8. UX 흐름
- 3단계 온보딩: ① 가입 즉시 키/몸무게/성별/나이 → 기본 체형 / ② 검진 결과지 OCR로 수치 잠금해제(한국 국가검진 표준 양식) / ③ 데일리 로그(아픈 부위, 사진).
- 수동 마커 보정 시 **마커 위치 예시 가이드**(정면/측면 라벨 그림) 제공.
- 하단 면책 고지 고정.

## 9. 기술 스택 & 빌드 노트 (실제 앱 기준)
- 프론트: React + Vite 권장(바닐라도 무방). 3D: **Three.js**. 자세: **@mediapipe/tasks-vision**(자동 추출이 기본 — 프로토타입의 수동 점은 fallback으로만).
- 상태/저장: **localStorage 또는 서버 DB 정상 사용**(프로토타입의 "금지"는 claude.ai 한정 제약이었음).
- 백엔드: AI 호출 프록시(키 보관) + 사용자 데이터 저장. OCR(검진 결과지)은 후속.
- 차트/표/스타일 자유. 다크 "바이오/스캔" 톤 유지 권장.

## 10. 현재 산출물(프로토타입) — 참고 입력
- `bodyscan.html`: 근골격/내과 2탭 분리, 자세(정면/측면) + 근육 맵(앞/뒤, 해부 톤 SVG), 장기 시각화, 탭별 리포트 + AI 버튼, 마커 위치 가이드.
- `anatomy3d-viewer.html`: Three.js로 `.glb` 로드 + 근육 이름 기반 `applyResult` 색칠 — **6번 Z-Anatomy 연동의 뼈대.**
- 목표: 이 둘을 통합하고, 근육 맵을 SVG → Z-Anatomy 3D로 교체/고도화.

## 11. 정직한 제약 (꼭 인지)
- 사진급 사실성은 **라이선스 에셋(BioDigital 등, 개발자 API 유료)** 이 필요. **Z-Anatomy(무료·오픈소스)가 최고 수준의 무료 옵션.**
- **Human Anatomy Atlas / Visible Body**는 폐쇄형 소비자 앱이라 모델 추출·임베드 불가(라이선스 위반). 자산 소스로 쓰지 말 것. (그쪽 콘텐츠가 필요하면 BioDigital 개발자 API 별도 계약.)
- Z-Anatomy의 Blender → `.glb` 변환은 **사람이 수동으로** 수행(코드 자동화 대상 아님).

## 12. 권장 빌드 순서
1. 앱 골격(2탭 + 공통 성별) + 체형/검진 입력 + 규칙 기반 리포트.
2. 장기 시각화(수치 → 색/이펙트).
3. **MediaPipe 자동 자세 분석**(정면/측면) + 지표/근육 규칙.
4. **Three.js + Z-Anatomy `.glb` 3D 근육 맵 + `applyResult` 연결**(핵심).
5. AI 리포트(백엔드 프록시).
6. 온보딩 3단계 + OCR.
